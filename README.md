# 🤖 미니 Transformer로 배우는 번역 모델 (Colab 실습)

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/unicorn-campus/voc-translate/blob/main/transformer_translate_colab.ipynb)

한국어 → 영어 번역을 하는 **아주 작은 Transformer**를 부품부터 직접 조립하며, 트랜스포머가 문장을
어떻게 이해하고(인코딩) 답을 어떻게 만들어내는지(디코딩)를 **눈으로 확인**하는 Colab 실습입니다.
`nn.Transformer`나 HuggingFace를 쓰지 않고 Q/K/V·스케일링·softmax·마스킹을 손으로 구현해
**동작 원리**를 보이도록 만들었습니다. GPU가 필요 없고, 학습은 몇 초면 끝납니다.

> 대상: 파이썬·딥러닝 **완전 초급자** · 실습 시간 **3.5시간** · 개인평가 반영
> (이 예제는 `start-here/hands-on/llm`의 로컬 실행 버전을 Colab 실습용으로 재구성한 것입니다.)

## 🎯 이 실습에서 배우는 6가지

1. **모델 학습 방법 이해** — 모델이 "틀린 만큼 조금씩 고쳐"(loss↓) 배우는 원리
2. **인코딩① 토큰화·임베딩·위치 인코딩** — 문장을 숫자로 바꾸고 순서 정보를 더하기
3. **인코딩② Multi-Head Attention** — 여러 관점으로 문맥을 파악
4. **인코딩③ 잔차+정규화+FFN** — 이해를 더 풍성하게 (인코더 블록)
5. **디코딩① 순차 응답 생성** — 한 단어씩, 커닝 없이(Masked), 원문 참고(Cross)
6. **디코딩② 잔차+정규화+FFN** — 응답 생성을 더 풍성하게 (디코더 블록)

각 항목은 **개념 비유 → (빈칸 채우기) → 즉시 결과 확인**으로 구성되며, 마지막 **7. 캡스톤**에서 완성한
부품을 조립해 실제로 학습·번역합니다.

## 📖 학습과정별 실습 방법

노트북을 위→아래 순서로 실행하며 진행합니다. 각 항목의 **기대 출력**이 실제로 나타나는지 꼭 확인하세요.

### 1. 모델 학습 방법 이해 (약 10분)
1. **1️⃣ 섹션** 셀 실행 → "학습 = 정답과 얼마나 다른지(loss)를 보고 조금씩 고치기" 개념 확인
2. 즉시 출력: 장난감 예제의 **loss 숫자** 하나 (학습이 진행되면 0에 가까워짐)
3. 실제 loss 곡선은 **7 캡스톤**에서 산출됩니다(부품을 모두 완성한 뒤 학습 실행).

### 2. 인코딩① 토큰화·임베딩·위치 인코딩 (약 25분)
1. **2️⃣ 섹션**에서 토큰화·임베딩 결과(토큰 id, `(1, 3, 64)` shape) 확인
2. **위치 인코딩**: 드롭다운에서 sin/cos 배치를 선택(객관식 1) → `✅ 위치 인코딩 정답!`
3. 기대 출력: **위치 인코딩 파도무늬(sin/cos) 히트맵**

### 3. 인코딩② Multi-Head Attention (약 50분)
1. **3️⃣ 섹션**에서 `Attention(Q,K,V)=softmax(QKᵀ/√dₖ)·V` 를 **직접 완성**(주관식 2) → `✅ Self-Attention 공식 정답!`
2. 헤드 분할 방법을 드롭다운에서 선택(객관식 1) → `✅ 헤드 분할 정답!`
3. 기대 출력: `"나는 사과를 먹었다"`에서 **'먹었다'가 각 단어에 주목하는 비율 — 헤드 4개 막대그래프**

### 4. 인코딩③ 잔차+정규화+FFN (인코더 블록) (약 20분)
1. **4️⃣ 섹션**에서 잔차+정규화 순서를 드롭다운에서 선택(객관식 1) → `✅ 인코더 블록 정답!`
2. 기대 출력: EncoderLayer forward **출력 shape `(1, 3, 8)`**

### 5. 디코딩① 순차 응답 생성 (Masked + Cross-Attention) (약 30분)
1. **5️⃣ 섹션**에서 **미래를 가리는 causal mask**를 직접 완성(주관식 1) → `✅ 미래 가림 마스크 정답!`
2. 다음 단어 고르기(greedy)를 드롭다운에서 선택(객관식 1) → `✅ greedy 선택 정답!`
3. 기대 출력: **미래 가림 마스크(하삼각) 히트맵**. 실제 단어별 생성 로그는 7 캡스톤에서 산출.

### 6. 디코딩② 잔차+정규화+FFN (디코더 블록) (약 20분)
1. **6️⃣ 섹션**에서 디코더 블록의 3단(self→cross→ffn) 순서를 드롭다운에서 선택(객관식 1) → `✅ 디코더 블록 정답!`
2. 기대 출력: DecoderLayer forward **출력 shape `(1, 3, 8)`**

### 7. 캡스톤 — 조립·학습·번역 (약 25분)
1. **7️⃣ 섹션**에서 부품을 조립해 Transformer를 만들고 **학습 실행**
2. 기대 출력(학습과정 1): **loss 곡선**(선형/로그 스케일) — `epoch 1 ~49 → epoch 400 ~0.12`
3. 기대 출력(학습과정 5): **단어별 순차 생성 로그**와 **Cross-Attention 히트맵**
   - `고객이 쿠폰을 취소한다` → `the customer cancels the coupon` (학습 문장)
   - `상담원이 배송을 등록한다` → `the agent registers the delivery` (처음 본 조합도 일반화!)
   - `결제가 지연된다` → `the refund is delayed` (**오역 사례** — 아래 관찰 포인트 참고)
4. 마지막 `🎉` 메시지가 보이면 6개 항목 전체 완주입니다.

> 🌐 (참고) 원본 로컬 버전 `start-here/hands-on/llm` 에는 동일 과정을 브라우저로 보는 **웹 시각화**
> (`#training`·`#attention`·`#translate`)가 있습니다. Colab에서는 로컬 웹서버를 띄우기 번거로워
> 동일한 그림을 **matplotlib로 노트북 안에 바로** 그리도록 재구성했습니다.

## ⚠️ 한계 및 관찰 포인트 (한계도 배움입니다)

- 학습 데이터가 100문장(글로서리 조합 240개 중 일부)뿐이라 **번역 품질 데모가 아니라 Transformer 내부
  동작 원리를 보는 교육용**입니다.
- 토크나이저는 공백 분리(word-level)만 지원합니다. 실제 서비스에는 BPE·SentencePiece가 필요합니다.
- **오역 사례**: held-out 문장 `결제가 지연된다`(정답: the payment is delayed)를 `the refund is delayed`로
  오역합니다. 상태형 문장 조합이 학습에 드물게 노출돼(대상 8 × 상태술어 2 = 16개뿐) 목적어를 혼동한
  것으로, **데이터가 작을 때 일반화가 약해지는 경계**를 눈으로 확인하는 유용한 관찰입니다.
